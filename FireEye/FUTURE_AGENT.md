# FireEye Future Agent — Technical Documentation

## Overview

The Future Agent is the final stage of the FireEye three-stage analysis pipeline. It receives a scene image, YOLO object detections, the Stage 2 risk classification, and the Present Agent's assessment, then predicts **branching future scenarios** describing how the fire situation could evolve.

```
Image  ──►  Stage 1: YOLO Detection
                │
                ▼
            Stage 2: Risk Classification  (heuristic + LLM)
                │
                ▼
            Stage 3a: Present Agent  ──►  Stage 3b: Future Agent
                                                │
                                                ▼
                                         FuturePrediction
                                           ├─ scenarios[]
                                           ├─ overall_risk
                                           └─ recommendation
```

Source: `app/pipeline/llm_agents.py:predict_future()`


## Risk Levels Are Condition-Based, Not Time-Based

A common misconception is that the risk levels (low → medium → high → critical) represent a **temporal progression** — i.e., that a scene starts at "low" and escalates to "critical" over time. **This is not how the system works.**

Each risk level describes the **current physical state of the scene** at the moment of analysis:

| Risk Level | What the Scene Shows Right Now |
|------------|-------------------------------|
| **low** | Controlled, isolated flame; no spread pathway visible. |
| **medium** | Controlled flame with flammable materials visible; spread needs a specific trigger (displacement, prolonged exposure). |
| **high** | Any of: (a) large uncontrolled fire regardless of visible fuel targets, (b) active ember/spark dispersal visible, (c) controlled flame immediately adjacent to significant fuel. |
| **critical** | Any of: (a) at least one predicted scenario that is "certain" with severity "high" or "critical", (b) multiple simultaneous "likely" high-severity pathways, (c) fire already actively spreading. |

A scene that is **low** right now can only become **critical** if the physical conditions change — fuel is introduced, the flame grows, wind picks up, or containment is lost. The system does not predict that a low-risk scene will *eventually* become critical through time alone.

### How long between low and critical?

**There is no fixed time interval.** The transition depends entirely on how the physical scene evolves:

- A candle on an empty floor (low) could remain low indefinitely — or become critical in seconds if a gas can is placed next to it.
- A contained welding torch (low) near stacked wood (medium) could escalate to high/critical if embers begin dispersing due to wind.
- In the worst case, physical changes (wind gust, structural collapse, fuel spill) can cause a jump from low to critical in **seconds to minutes**.
- In benign conditions, a low-risk scene may never escalate at all.

The system is designed for **repeated real-time analysis**: each frame/image is independently classified. If conditions worsen between captures, the risk level will escalate accordingly. The value of the Future Agent is in flagging **what physical changes would cause escalation** and how likely those changes are.


## Future Agent Output Schema

Each call to the Future Agent produces a `FuturePrediction` (defined in `app/models/schemas.py`):

```
FuturePrediction
├── scenarios: list[FutureScenario]
│   └── FutureScenario
│       ├── scenario:      str    # Description of what could happen
│       ├── likelihood:    str    # "unlikely" | "possible" | "likely" | "certain"
│       ├── severity:      str    # Free-form severity assessment
│       └── time_horizon:  str    # When it could happen (e.g., "30 seconds", "2–5 minutes")
├── overall_risk: RiskLevel        # "safe" | "low" | "medium" | "high" | "critical"
└── recommendation: str            # Suggested action
```

### Likelihood Scale

| Likelihood | Meaning |
|-----------|---------|
| **unlikely** | Requires a specific accident (displacement, strong wind gust). |
| **possible** | Plausible given normal activity, but not actively occurring. |
| **likely** | A natural continuation of what is already visible in the scene. |
| **certain** | Already happening or physically inevitable. |

### The `time_horizon` Field

Each scenario includes a `time_horizon` — an estimate of **when that specific scenario could unfold** if the triggering conditions are met. This is a free-form string generated by the LLM (e.g., "within seconds", "1–3 minutes", "10–15 minutes if unattended").

This field answers: *"If this scenario's trigger occurs, how quickly would the consequences manifest?"* It does **not** predict when the trigger itself will occur.


## Design Principles

The Future Agent prompt is specifically engineered to avoid alarm inflation:

1. **Proportional assessment** — Open flames are normal in construction/industrial contexts (welding, cutting torches, candles). A flame alone is not a crisis.

2. **Physical mechanism required** — Each scenario must explain the specific trigger (direct contact, radiant heat, ember travel, displacement) needed for escalation. No hand-waving.

3. **Anchored in observation** — The agent differentiates between "actively occurring" (what the image shows) and "would require an unlikely accident" (hypothetical).

4. **Ember/spark rule** — Active ember or spark dispersal visible in the image is always treated as at least HIGH risk, because embers travel beyond the visible frame and can land on unknown materials.

5. **Focused scenarios** — The agent is instructed NOT to list every conceivable bad outcome. Only scenarios that the scene actually indicates are included.


## Escalation Drivers

The following physical factors drive escalation from one risk level to the next:

```
LOW  ──────────────────────────────────────────────►  CRITICAL

 ┌─────────────┐    ┌──────────────┐    ┌──────────────┐    ┌───────────────┐
 │   LOW       │    │   MEDIUM     │    │   HIGH       │    │   CRITICAL    │
 │             │    │              │    │              │    │               │
 │ Controlled  │───►│ + Fuel       │───►│ + Active     │───►│ + Multi-path  │
 │ isolated    │    │   visible    │    │   dispersal  │    │   cascade or  │
 │ flame       │    │   nearby     │    │   or large   │    │   already     │
 │             │    │              │    │   uncontrolled│    │   spreading   │
 └─────────────┘    └──────────────┘    └──────────────┘    └───────────────┘

Key transition triggers:
  low → medium    : Flammable material enters the scene / proximity decreases
  medium → high   : Flame grows uncontrolled, embers begin dispersing,
                    or fuel becomes immediately adjacent
  high → critical : Multiple high-severity pathways converge, or fire
                    is already actively spreading to new materials
```

## Pipeline Integration

The orchestrator (`app/pipeline/orchestrator.py`) calls the Future Agent as the final stage:

1. **Stage 1** — YOLO detects objects (fire, flame, smoke, gas cylinders, wood, etc.)
2. **Stage 2** — LLM classifies overall scene risk from the image + detections
3. **Stage 3a** — Present Agent describes current hazards and spatial relationships
4. **Stage 3b** — Future Agent predicts scenarios, assigns `overall_risk`, and recommends action

The Future Agent receives the Present Agent's output as input, ensuring its predictions are grounded in observed spatial relationships (e.g., "wood stack is 0.5m from flame") rather than re-interpreting the raw image independently.
